    2. Go to **Settings** -> **API**.
    3. Copy the **Project URL** and **Service Role/Anon Key**.
    4. **Create the Table**: Open the **SQL Editor**, paste the following, and click **Run**:
       ```sql
       CREATE TABLE messages (
         id BIGSERIAL PRIMARY KEY,
         created_at TIMESTAMPTZ DEFAULT NOW(),
         name TEXT,
         email TEXT,
         message TEXT
       );
       ```

## üíª Part 2: Local Development (Running on your Machine)

### Step 1: Backend Setup (The Engine)
1.  Open the `spark` folder in VS Code.
2.  Create a file named `.env` in the root `spark` folder:
    ```env
    GOOGLE_API_KEY=your_key_here
    SUPABASE_URL=your_url_here
    SUPABASE_KEY=your_key_here
    ```
3.  **Initialize the Environment**:
    ```powershell
    # Create the virtual environment
    python -m venv .venv
    
    # Activate the environment
    .\.venv\Scripts\Activate.ps1
    
    # Install dependencies
    pip install -r requirements.txt
    ```
4.  **Run the Backend**:
    ```powershell
    python app.py
    ```
    *You should see: "Running on http://127.0.0.1:5000"*

### Step 2: Frontend Setup (The Website)
1.  Open a **new terminal** window in VS Code.
2.  Navigate to the frontend folder:
    ```powershell
    cd spark-ai-hub-main
    ```
3.  Create a `.env` file inside `spark-ai-hub-main`:
    ```env
    VITE_API_URL=http://localhost:5000
    ```
4.  **Launch the Frontend**:
    ```powershell
    npm install
    npm run dev
    ```
5.  Open the URL provided (default is `http://localhost:8080` or `8082`).

## üßπ Part 4: Maintenance & Cleanup

### Deleting the `.venv`
If you are done with local testing and have pushed your code to the cloud, you can safely delete the `.venv` folder to save space (it can be over 500MB).
- **To delete**: Simply right-click the `.venv` folder and click **Delete**.
- **To Rerun later**: Follow **Part 2, Step 1.3** again to recreate it.

### Troubleshooting Common Errors
- **"Python not found"**: Ensure Python is in your system PATH. Restart VS Code.
- **"CORS Error"**: Ensure the backend `app.py` has `CORS(app)` enabled and your frontend URL is allowed.
- **"Port already in use"**: If Port 5000 is taken, change it in `app.py` at the bottom: `app.run(port=5001)`.
